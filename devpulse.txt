## 5) Teach-Kel primer (the concepts YOU are learning)
Plain-English so you can teach it while you learn it:

- **API** = a way for your code to ask another company's server for data. "Hey Open-Meteo, what's the weather?" It answers.
- **`fetch(url)`** = the JavaScript command that sends that request over the internet.
- **JSON** = the format the answer comes back in. It looks like labeled boxes: `{ "temperature": 72 }`. You read it with dots: `data.temperature`.
- **`async` / `await`** = "wait for the internet to answer before moving on." Network calls take time; `await` pauses politely instead of breaking.
- **DOM update** = putting the answer onto the page (`setCard(...)` swaps the card's text for real data).
- **`try / catch`** = a safety net. If an API fails, catch the error and show "couldn't load" instead of crashing.
- **`setInterval`** = "do this again every X seconds" (how the crypto card refreshes itself).
- **Deploy** = putting your file on the internet so it has a real URL anyone can open. Netlify does it by drag-and-drop.

**Your one-liner for the room:** "An API is a waiter. `fetch` is you ordering. JSON is the plate that comes back. We just put what's on the plate onto the screen."
