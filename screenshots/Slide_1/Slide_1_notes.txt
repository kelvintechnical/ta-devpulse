What is a .cshtml file?
A .cshtml file is an HTML page that can have C# code mixed into it. The cs part stands for C#, and html is the markup. So instead of writing plain static HTML, you can write HTML that also runs little snippets of C# — for example to insert a variable's value, loop over a list, or make a decision — right in the middle of the markup.
Think of it as: HTML is the skeleton, and .cshtml lets you inject C# "muscles" into that skeleton before the page gets sent to the browser.
Why did Index.cshtml appear when I ran dotnet new webapp?
dotnet new webapp is a template command — it scaffolds a starter Razor Pages project for you. Every Razor Pages project needs at least one page to show when someone visits the site, so the template automatically creates a default homepage called Index.cshtml inside the Pages folder.
The name Index is special: by convention, when someone visits your site's root URL (like localhost:5000/), Razor Pages automatically looks for and serves Index.cshtml. That's why it "just appeared" and "just worked" without you wiring up any routing yourself.

**How is `.cshtml` different from a normal `.html` file?**

A plain `.html` file is static — whatever text is written in it is exactly what the browser receives, every single time, no matter what.

A `.cshtml` file gets **processed by the server first**. Before it's sent to the browser, the server runs any C# code embedded in it (using the `@` symbol) and replaces that code with actual output — like inserting today's date, a variable's value, or the results of a loop. The browser never sees the C#; it only receives the final plain HTML that comes out the other end.

So: `.html` = fixed content. `.cshtml` = a template that gets filled in dynamically each time the page loads.

What are Razor Pages in one simple sentence?
Razor Pages is a way of building a web app where each page of your site is its own self-contained pair of files — a .cshtml file for the markup/UI and a matching .cshtml.cs file for the C# logic behind it — instead of routing everything through one giant central controller.
